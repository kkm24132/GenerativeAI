
## RAG and Vector Databases

I would encourage to gather understanding on **RAG frameworks**:
  - LangChain: https://www.langchain.com/
  - LlamaIndex: https://www.llamaindex.ai/

**Key Vector Databases**
  - [ChromaDB](https://www.trychroma.com/) - Fast, serverless, and scalable infrastructure supporting vector, full-text, regex, and metadata search. Built on object storage and trusted by millions of developers. Open-source Apache 2.0.
  - [Qdrant](https://qdrant.tech/) - Qdrant helps you build the AI retrieval you want. Ship high performance, full-feature vector search at any scale and with any deployment model.
  - [Pinecone](https://www.pinecone.io/) - Pinecone is a fully managed vector database built for AI. Writes are instantly searchable, indexing is automatic, and queries stay fast at any scale.
  - [Weaviate](https://weaviate.io/) - is an open-source vector database designed for AI and semantic search applications. It helps store and retrieve embeddings generated from LLMs, enabling capabilities such as Retrieval-Augmented Generation (RAG), similarity search, recommendation systems, and context-aware AI applications at scale.
  - [Milvus](https://milvus.io/) - Milvus is an open-source vector database built for GenAI applications. Install with pip, perform high-speed searches, and scale to tens of billions of vectors with minimal performance loss.

For **Embedding Models**, start with the following:
  - [OpenAI Embeddings](https://developers.openai.com/api/docs/guides/embeddings)
  - [Voyage AI](https://www.voyageai.com/)
  - [Hugging Face Embeddings](https://huggingface.co/blog/getting-started-with-embeddings)

Good reference - to check how to choose the [Best Embedding Model for RAG in 2026: 10 Models Benchmarked](https://milvus.io/blog/choose-embedding-model-rag-2026.md)
